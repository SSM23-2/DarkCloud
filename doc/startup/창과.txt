1. 주제

Android + Cloud Service

2. 선정배경

Cloud Service란? -> 필요있나?.

현재의 Cloud Service -> 진정한 Cloud Service가 아니다.
	사용자들 설문조사 첨부(재석이형) -> 시작문서에? 일단...물어보고
	Google Drive(동기화), Naver N드라이브(마찬가지)

현재의 Cloud System -> 동기화 방식, 결국 PC나 안드로이드에 저장된 후 파일을 쓰게 된다.

그렇기 때문에 이 프로그램을 개발할 것이다.


3. 개발목적 ----------------------------------- 흠흠흠흠흠 굳이 선정배경이랑 개발목적이랑 두개 나눌필요?

 Android에서 Cloud Service 이용 - 서버에 저장되고, 바로 Android에서 확인가능

 ex) 사진을 찍으면 바로 서버에 전송해서 서버에 파일이 저장됨, Android에서 확인 가능
 

 -> 충분한 데이터와 안정된 네트워크 서비스를 가진다는 가정 하에 거의 무한대의 용량 사용 가능

5. 개발방법

 1. Cloud Server - 현재 나와있는 오픈소스는 오픈스택, OwnCloud 두 개 존재, 구현하려면 너무 시간이 오래 걸리고, 오픈소스를 이용하면 너무 쉬워져서 본래의 개발 목적 및 우리의 역량 발전에도 도움이 안됨. 따라서 소스를 참고하되 우리가 구현하는 방식으로 개발
